GitHub
------
1. On github.com: New repository (e.g. eight-elements), empty, no README.
2. In this folder: git remote set-url origin https://github.com/motiv8advisors77/eight-elements.git
3. git push -u origin main

Supabase (schema)
-----------------
1. New project in Supabase dashboard.
2. SQL Editor: paste and run supabase/migrations/20260214120000_client_plans.sql (skip if table already exists).

Supabase (auth URLs)
--------------------
Authentication > URL configuration:
- Site URL: your production URL after Vercel deploy (e.g. https://your-app.vercel.app)
- Redirect URLs: http://localhost:3000/auth/callback AND https://your-app.vercel.app/auth/callback

Vercel
------
1. vercel.com → Log in → Add New… → Project → Import Git Repository → pick motiv8advisors77/eight-elements (or your org/repo).
2. Framework Preset: Next.js (auto). Root Directory: ./ (repo root). Build: npm run build or pnpm build (Vercel picks pnpm if pnpm-lock.yaml exists).
3. Environment Variables — add for Production (and Preview if you want preview deploys to hit Supabase):
   NEXT_PUBLIC_SUPABASE_URL = https://YOUR_REF.supabase.co   (project URL only; NO /rest/v1/)
   NEXT_PUBLIC_SUPABASE_ANON_KEY = (anon public key from Supabase → Project Settings → API)
4. Deploy. Copy the production URL (e.g. https://eight-elements-xxx.vercel.app).
5. In Supabase → Authentication → URL configuration: set Site URL to that production URL; add https://YOUR_VERCEL_HOST/auth/callback to Redirect URLs (keep localhost too for dev).

Local
-----
1. cp .env.example .env.local  (Windows: copy .env.example .env.local)
2. Fill NEXT_PUBLIC_* from Supabase.
3. pnpm install && pnpm dev
